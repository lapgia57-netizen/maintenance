Webhook là gì?

Giải thích siêu đơn giản cho người mới bắt đầu

Nói đơn giản nhất

Webhook giống như "tin nhắn tự động" mà một ứng dụng gửi cho ứng dụng khác ngay khi có chuyện gì đó xảy ra, thay vì phải hỏi liên tục "có gì mới không?".

Người ta hay gọi Webhook là "Reverse API" hay "API ngược" đấy!

So sánh dễ hình dung nhất

Tiêu chí API truyền thống (Polling) Webhook
Ai khởi tạo Bạn (ứng dụng của bạn) Ứng dụng kia (server bên kia)
Cách lấy dữ liệu Hỏi liên tục mỗi 1–5–10 giây Được "đẩy" về ngay lập tức
Tốn tài nguyên server Rất tốn (nhiều request vô ích) Tiết kiệm cực kỳ
Độ realtime Chậm (phụ thuộc khoảng cách hỏi) Gần như tức thì
Ví dụ đời thực Bạn F5 trang Facebook liên tục xem có like mới Facebook tự đẩy thông báo khi ai đó like bài của bạn

Ví dụ thực tế rất phổ biến

Có người thanh toán qua Stripe Stripe → Website của bạn → Tự động gửi email cảm ơn, cập nhật đơn hàng
Ai đó push code lên GitHub GitHub → Server CI/CD → Tự động build/test/deploy website mới
Có đơn hàng mới trên Shopee/Tiki Shopee → Hệ thống kho → Tự động trừ tồn kho, chuẩn bị đóng gói
Tin nhắn mới đến số Twilio Twilio → Ứng dụng chat → Hiển thị tin nhắn realtime

Cách hoạt động cơ bản (chỉ 3 bước)

  1. Bạn tạo một URL công khai (webhook endpoint)
    Ví dụ: https://your-site.com/webhook/payment
  2. Đăng ký URL này với dịch vụ (Stripe, GitHub, Facebook...)
  3. Khi sự kiện xảy ra → dịch vụ gửi HTTP POST kèm JSON đến bạn
{ "event": "payment.success", "order_id": "ORD12345", "amount": 1490000, "customer_email": "khach@gmail.com", "timestamp": "2025-12-27T14:35:22Z" }

Server của bạn chỉ cần nhận request → xử lý → trả về 200 OK là xong!

Tóm lại siêu ngắn:
Webhook = "Ê, có chuyện này nè!" → đẩy thẳng dữ liệu đến bạn ngay lập tức
→ Tiết kiệm, nhanh, realtime, hiện đại!

Hầu hết dịch vụ lớn đều hỗ trợ webhook (Stripe, GitHub, Shopify, Discord, Zoom...)
